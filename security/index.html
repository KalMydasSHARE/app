Protections actives

🛡️

Anti-slippage

Tous les swaps AMM ont un minimum de 5% de protection contre la manipulation de prix.

🔒

Anti-rug pull

Les retraits de liquidité LP sont limités à 25% maximum par période de 24 heures.

⚡

Anti-MEV

Les fonctions de distribution de surplus sont restreintes au owner (pas de front-running possible).

🔄

Anti-race condition

La prévente vérifie que le round n'a pas changé pendant la transaction.

📊

High-Water Mark

Les frais de performance ne sont prélevés que sur les nouveaux profits (pas de double facturation).

🔐

ReentrancyGuard

Toutes les fonctions critiques utilisent le guard OpenZeppelin contre les attaques de réentrance.

Résultats d'audit

Audit interne #1 — 14 mars 2026Audit interne #2 — 14 mars 2026

Analyse statique complète de 12 contrats Solidity couvrant le slippage, la réentrance, le contrôle d'accès, la manipulation d'oracle, les conditions de course et la protection contre le rug-pull. Toutes les vulnérabilités critiques et hautes ont été corrigées.

CRITIQUE — 2 corrigées

Slippage 0 sur les swaps AMM (KalPool, BondingCurve, BuybackBurner) — corrigé : 5% minimum
Underflow réserve dans reportLoss — corrigé : require strict

HAUTE — 5 corrigées

Front-running distributeSurplus — corrigé : onlyOwner
Race condition prévente — corrigé : revert si round avance pendant TX
Rug-pull LP (retrait illimité) — corrigé : max 25% par 24h
Centralisation owner — atténué : Gnosis Safe 2/3 prévu pour mainnet
stakeLPFor sans événement — corrigé : event + validation adresse

MOYENNE — 5 corrigées

Slippage prévente addLiquidity — corrigé : 5%
maxOperatorWithdrawAmount par défaut trop élevé — corrigé : 100x minDeposit
targetKALBalance non initialisé — corrigé : 1000 KAL en constructeur
Ordre harvest airdrop — corrigé : documentation NatSpec
Vérification balance airdrop — corrigé : require avant transfert
